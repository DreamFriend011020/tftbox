# 현재 프로젝트 이슈 및 해결 가이드

## 1. 랭크 정보(티어/LP) 미표시 현상
- **증상**: 소환사 검색 시 우측 상단에 티어 정보 대신 "랭크 정보 없음" 또는 "?" 아이콘이 표시됨.
- **원인**: Riot API가 특정 계정(장기간 미접속, 데이터 이관 등)에 대해 `id` (Encrypted Summoner ID)를 반환하지 않는 경우가 있음.
- **현재 조치**: 서버(`server.js`)에 3중 안전장치 적용 완료.
    1. TFT API 조회 시도
    2. 실패 시 LoL API 조회 시도 (Fallback 1)
    3. 실패 시 최근 매치 기록에서 ID 역추적 시도 (Fallback 2)
- **참고**: 위 3가지 방법으로도 ID를 찾을 수 없는 경우(예: 이번 시즌 게임 기록이 아예 없는 경우)에는 기술적으로 랭크 정보를 가져올 방법이 없으므로, "랭크 정보 없음"으로 표시되는 것이 정상 동작임.

## 2. 일부 아이템 이미지 미출력
- **증상**: 특정 아이템(찬란한 아이템, 유물 등)의 아이콘이 엑스박스로 나오거나 빈 칸으로 표시됨.
- **원인**: Riot의 공식 데이터 파일(`item.json`, `tft-item.json`)에 최신 시즌의 특수 아이템 정보가 누락되어 있어, 아이템 ID를 이미지 파일명으로 변환하지 못하는 경우가 있음.
- **현재 조치**:
    - `summoner.js`에서 아이템 데이터를 직접 로드하여 매핑 시도.
    - 이미지 로드 실패 시 `ddragon` -> `tft-item` -> `communitydragon` 등 5단계 경로를 순차적으로 탐색하도록 구현.
- **해결 방법**: 로컬에 이미지를 다운로드해두면 가장 확실함. 터미널에서 아래 명령어를 실행하여 이미지 최신화 권장.
    ```bash
    node download_images.js
    ```

## 3. 서버 재시작 필요 (운영 이슈)
- **증상**: 코드를 수정했는데도 에러(`Summoner ID is missing` 등)가 계속 발생하거나 기능이 동작하지 않음.
- **원인**: `server.js` (백엔드) 코드는 수정 후 **반드시 서버를 재시작**해야 변경 사항이 반영됨.
- **해결 방법**: 코드를 수정할 때마다 터미널에서 `Ctrl + C`로 서버를 끄고 `node server.js`로 다시 시작해야 함.
